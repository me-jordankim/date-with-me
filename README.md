# 나랑 데이트할래? 💕

데이트 신청 웹서비스. 모바일·데스크탑 모두 지원하며, **백엔드나 DB 없이** 완전히 정적으로 동작합니다.

## 기능

- **데이트 신청 플로우** (`index.html`)
  1. "나랑 데이트할래?" — "응/아니" (아니 버튼은 데스크탑에선 마우스를 피하고, 모바일에선 순간이동)
  2. 캘린더에서 날짜 선택
  3. 같이 하고 싶은 활동 다중 선택
  4. 고른 활동별 세부 옵션 선택
  5. 완성 요약 — 텍스트 복사 / 이미지 저장 / **답장 링크** 생성
- **만들기 페이지** (`create.html`) — 이름·문구·활동 목록을 커스터마이징해 초대 링크 생성
- **답장 링크** — 상대가 고른 결과가 링크에 담겨 돌아옴 (만든 사람이 열면 결과 확인)

모든 설정과 답장은 **URL 해시에 압축 인코딩**되어 저장되므로 서버가 필요 없습니다.

## 파일 구성

| 파일 | 설명 |
|------|------|
| `index.html` | 데이트 신청 + 답장 확인 화면 (단일 파일) |
| `create.html` | 초대 링크 만들기 페이지 |
| `favicon.svg` | 하트 파비콘 |
| `vercel.json` | 정적 배포 설정 (clean URLs, 캐싱/보안 헤더) |

빌드 도구·의존성이 전혀 없습니다.

## Vercel 배포

### 방법 1) 대시보드에서 Import (권장)
1. 이 저장소를 GitHub에 푸시 (이미 되어 있음)
2. [vercel.com/new](https://vercel.com/new) → 저장소 Import
3. Framework Preset은 **Other**, Build Command·Output 디렉터리는 **비워두기** (정적 파일 그대로 서빙)
4. **Deploy** 클릭 → `https://<프로젝트>.vercel.app` 완성

### 방법 2) Vercel CLI
```bash
npm i -g vercel   # 최초 1회
vercel            # 프리뷰 배포
vercel --prod     # 프로덕션 배포
```

## 로컬에서 열기

각 HTML 파일을 브라우저로 바로 열면 됩니다. 페이지 간 이동과 링크 생성 모두 `file://`에서도 동작합니다.

## 브라우저 지원

링크 압축에 브라우저 내장 `CompressionStream`(deflate-raw)을 사용합니다. 미지원 구형 브라우저에서는 자동으로 비압축(조금 더 긴) 링크로 동작합니다.
